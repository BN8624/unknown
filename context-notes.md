## TASK_001 결정

- 엔진: 설치된 Godot 4.7.stable 사용 (`C:\Users\USER\godot-engine\Godot_v4.7-stable_win64.exe`).
- 임시 도형은 별도 에셋 없이 코드에서 `ColorRect`로 생성. 용병/적/체력바/배경 줄무늬 모두 동일 방식.
- 상태 머신은 범용 프레임워크 없이 `enum { WALK, COMBAT, DEAD }`와 변수만으로 최소 처리.
- 유닛은 별도 클래스/씬으로 분리하지 않고 `Dictionary`(merc/enemy)로 보관. 단일 화면·단일 적 종류라 과분리 회피.
- 피해 계산은 `피해량 = 공격력`으로 단순화하고 무작위를 쓰지 않아 같은 조건에서 같은 결과가 나오게 함. (방어력·치명타·확률 효과는 TASK_001 범위 밖)
- 임시 수치는 TASK_001 카드 값(용병 100/10/1.0초, 적 30/4/1.5초)을 사용. `UNKNOWN.md` 15·16장의 정식 임시 수치(용병 120/12, 늑대 32/5 등)와 다른데, 이는 카드가 명시한 첫 루프 검증용 값이라 카드를 우선함. 성장 시스템 붙일 때 `UNKNOWN.md` 값으로 맞춘다.
- 타격 표현은 카드가 요구한 "2가지 이상" 중 3가지 사용: 공격 시 전진 모션, 피격 시 밝아짐, 체력 바 감소.
- 검증: 헤드리스 실행이 불가능하면 자동 검증을 못 하므로, `-- --verify` 인자를 받으면 `Engine.time_scale=8`로 가속하고 적 6마리 처치 후 자동 종료하며 로그를 남기도록 함. 수치는 그대로, 시간만 가속이라 결과는 동일.
