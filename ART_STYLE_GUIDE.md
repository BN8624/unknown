# ART_STYLE_GUIDE.md — 아트 화풍 정본

이 문서는 `unknown`의 2D 컷아웃 아트 화풍 정본이다. 모든 신규 캐릭터·적·외형 에셋은 이 가이드와 승인된 기준 이미지를 함께 참조해 만든다. 승인 출처는 GitHub Issue #3(TASK 015) 댓글이다.

> 아트 방향(2D 컷아웃, AI 생성 분리 파츠)과 슬라이스 범위의 정본은 `VERTICAL_SLICE.md`다. 이 문서는 그 안에서 "화풍·비율·크기·승인 절차"만 고정한다.

## 기준 이미지 (제작 참고용, 런타임 비사용)

조립된 기준 디자인 1장씩을 종족·등급의 화풍 기준으로 삼는다. 게임 런타임에는 로드하지 않으며 파츠 제작 참고용으로만 보관한다.

| 대상 | 경로 | 상태 |
|---|---|---|
| 용병(인간 기준) | `assets/ref/merc_reference.png` | 화풍 승인됨(Issue #3). 파일은 저장소 추가 예정 |
| 굶주린 늑대(몬스터 기준) | `assets/ref/wolf_reference.png` | 화풍 승인됨(Issue #3). 파일은 저장소 추가 예정 |

- 새 캐릭터를 만들 때마다 위 기준 이미지 + 이 문서 + 같은 종족/등급의 기존 승인 에셋을 함께 참조한다. **프롬프트만으로 독립 생성하지 않는다.**

## 공통 화풍

- 2D 판타지 모바일 게임 아트, **완전한 측면 시점**.
- 깨끗하고 비교적 **굵은 외곽선**(축소 표시 후에도 유지될 굵기).
- 단순한 **셀 셰이딩**(과한 그라데이션·질감 금지).
- 세미 카툰 스타일(실사풍 아님).
- 작은 모바일 화면에서도 읽히는 **명확한 실루엣**.
- 지나치게 세밀한 질감·장식 금지(축소 시 사라지는 디테일은 제거).
- **배경보다 캐릭터의 명도·대비가 높아야 함**(배경은 저채도로 가라앉히고 캐릭터가 떠 보이게).

## 팔레트

- 저채도 **갈색·회색·흙색** 중심(황량한 변경 교역로 분위기).
- 배경: 탁한 저채도(흙길·메마른 덤불·흐린 하늘). 캐릭터·공격 효과·피해 숫자·UI는 항상 배경보다 밝게.
- 구체 색은 승인된 기준 이미지를 정본으로 한다(현재 코드의 임시 도형 색은 참고값일 뿐 화풍 기준 아님).

## 외곽선·명암

- 외곽선: 굵고 일정, 캐릭터 내부 디테일보다 실루엣 경계를 우선.
- 명암 단계: 단순한 2~3단(베이스 + 그림자 + 선택적 하이라이트).
- **광원 방향은 모든 에셋에서 통일**한다. 방향의 정본은 승인된 기준 이미지이며, 신규 에셋은 그 광원·명암 단계를 그대로 따른다(임의로 바꾸지 않는다).

## 비율

**인간 캐릭터(용병 기준)**
- 성인 인간 비율 유지. 단 모바일 가독성을 위해 머리·무기는 약간 강조 가능.
- 팔·다리·머리·몸통·무기가 명확히 구분. 무기는 몸과 **분리 가능한 실루엣**.
- 컷아웃 기준 자세는 팔다리·무기가 과도하게 겹치지 않는 중립 자세.
- 등급(외형 승급)이 올라가도 같은 얼굴·체형·외곽선 스타일 유지.

**몬스터**
- 머리·몸통·다리·꼬리가 명확히 구분.
- 종류는 색만이 아니라 **실루엣·체형**으로 구분.
- 털·표면 묘사는 기준 이미지보다 단순화(실제 런타임 에셋 기준).

## 크기 기준 (현재 코드 프로필 = 표시 기준 박스)

값은 `EnemyProfiles.gd`의 `size`(화면 표시 px 기준). 컷아웃은 이 박스를 기준으로 정렬하고, 다리·꼬리·무기는 박스 밖으로 나가도 된다. **일반 적 < 엘리트 < 보스**가 실루엣 크기·무게감으로 드러나야 한다.

| 등급 | 대상 | 기준 박스(px) |
|---|---|---|
| 플레이어 | 용병 | 60 × 90 |
| 일반 적 | 고블린 | 42 × 64 |
| 일반 적 | 굶주린 늑대 | 78 × 44 |
| 일반 적 | 방패병 | 66 × 96 |
| 엘리트 | 오우거 징수꾼 | 108 × 132 |
| 보스 | 철퇴의 브루노 | 132 × 156 |

- 엘리트·보스는 일반 적보다 **크고 무겁게**. 보스가 가장 큰 실루엣.
- 박스 크기·위치는 전투 판정(거리·체력바) 기준이므로 임의 변경 금지(변경 필요 시 기획 보고).

## 파츠 제작 원칙

- 기준 디자인과 파츠 결과의 **얼굴·비율·색상이 달라지면 안 됨**.
- 회전 관절 주변에는 가려진 영역을 충분히 그려 회전 시 **빈틈 방지**.
- 모든 파츠는 투명 PNG. 같은 캐릭터의 파츠는 **동일한 공유 캔버스** 사용(캔버스·피벗 좌표는 Issue #3 에셋 계약 v2 참조).
- 축소 표시 시 사라지는 작은 장식은 제거. 외곽선 굵기는 축소 후에도 유지될 정도로.

## 금지 사례

- 실사풍, 고해상도 사진풍.
- 고프레임 픽셀 아트(캐릭터당 수십 프레임).
- 과한 질감·장식, 축소 시 뭉개지는 디테일.
- 배경보다 어둡거나 대비가 낮아 묻히는 캐릭터.
- 가로·세로 비율을 다르게 늘린 왜곡.
- 완성 한 장을 나중에 억지로 잘라 만든 파츠(처음부터 컷아웃 리깅용으로 제작).
- 프롬프트만으로 기준 이미지 없이 독립 생성.

## 신규 에셋 승인 절차

1. 조립된 기준 디자인 생성(같은 종족/등급의 승인 에셋·이 문서 참조).
2. 사용자 승인.
3. 컷아웃 파츠 분리.
4. 공유 캔버스·피벗 좌표 적용(Issue 에셋 계약).
5. Godot 조립 확인(피벗·z순서·정렬).
6. 아이폰 가독성 확인.

기준 디자인과 파츠 결과가 어긋나면 파츠 단계로 되돌린다. 최종 미술 취향·가독성 승인은 각 아트 TASK의 아이폰 확인에서 한다.
