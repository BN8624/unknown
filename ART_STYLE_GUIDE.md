# ART_STYLE_GUIDE.md — 아트 화풍 정본

이 문서는 `unknown`의 아트 화풍·제작 방식 정본이다. 모든 신규 캐릭터·적·외형 에셋은 이 가이드를 따른다. 승인 출처는 GitHub Issue #3(TASK 015) 댓글이다.

> 슬라이스 범위·방향의 정본은 `VERTICAL_SLICE.md`다. 이 문서는 그 안에서 "화풍·비율·크기·제작 방식·승인 절차"를 고정한다.

## 제작 방식 (정본) — SD 전신 스프라이트

컷아웃 파츠 방식은 **폐기**했다(AI 생성으로 동일 캐릭터의 파츠 비율·관절 연결을 안정적으로 유지하기 어렵고, 조립 결과에서 머리·몸·팔다리 비율이 무너지는 문제가 확인됨, Issue #3).

새 정본은 다음과 같다.

```text
캐릭터:
  SD 전신 투명 PNG 1장
  → Godot Sprite2D 하나로 적용
  → 코드 기반 대기·공격·피격·죽음 연출(파츠 분리·관절 애니메이션 없음)
배경:
  하늘·지면 2레이어 PNG
  → 서로 다른 속도로 가로 스크롤(하늘 느리게, 지면 빠르게)
```

- 캐릭터는 파츠로 분리하거나 관절 애니메이션을 적용하지 않는다.
- 애니메이션은 전신 Sprite2D 하나에 트윈(이동·확대·회전·플래시)으로 준다. 기존 전투 판정·타이밍은 변경하지 않고 시각 연출만 연결한다.

## 런타임 에셋

| 대상 | 경로 | 비고 |
|---|---|---|
| 용병 | `assets/characters/mercenary.png` | SD 전신, 오른쪽(측면/측면에 가까운) 자세, 투명 PNG |
| 굶주린 늑대 | `assets/characters/wolf.png` | SD 전신, 왼쪽(용병을 바라봄), 투명 PNG |
| 배경 하늘 | `assets/bg/bg_sky.png` | 흐린 하늘·먼 산, 화면 0.5배 표시, 가로 반복 |
| 배경 지면 | `assets/bg/bg_ground.png` | 먼지 낀 교역로(마차 자국·돌·마른 풀), 화면 0.5배 표시, 가로 반복 |

- **캐릭터 PNG는 반드시 투명 배경(알파 채널 포함)**. 불투명 배경은 게임에서 사각형으로 보여 실패.
- 용병·늑대는 같은 외곽선·명암·색감 계열을 사용한다. 새 캐릭터를 만들 때 기존 승인 캐릭터와 이 문서를 함께 참조하고, 프롬프트만으로 독립 생성하지 않는다.

## 공통 화풍

- 2D 판타지 모바일 게임 아트, SD(머리·몸 강조) 전신, 측면 또는 측면에 가까운 자세.
- 깨끗하고 비교적 **굵은 외곽선**(축소 표시 후에도 유지될 굵기).
- 단순한 **셀 셰이딩**(과한 그라데이션·질감 금지).
- 세미 카툰 스타일(실사풍 아님).
- 작은 아이폰 화면에서도 읽히는 **명확한 실루엣**.
- 지나치게 세밀한 질감·장식 금지(축소 시 사라지는 디테일은 제거).
- **배경보다 캐릭터의 명도·대비가 높아야 함**(배경은 저채도로 가라앉히고 캐릭터가 떠 보이게).

## 팔레트

- 저채도 **갈색·회색·흙색** 중심(황량한 변경 교역로 분위기).
- 배경: 탁한 저채도(흙길·메마른 덤불·흐린 하늘). 캐릭터·공격 효과·피해 숫자·UI는 항상 배경보다 밝게.

## 외곽선·명암

- 외곽선: 굵고 일정, 캐릭터 내부 디테일보다 실루엣 경계를 우선.
- 명암 단계: 단순한 2~3단(베이스 + 그림자 + 선택적 하이라이트).
- **광원 방향은 모든 에셋에서 통일**한다. 신규 에셋은 기존 승인 캐릭터의 광원·명암 단계를 그대로 따른다(임의로 바꾸지 않는다).

## 비율

**인간 캐릭터(용병 기준)**
- SD 비율(성인 형태를 유지하되 모바일 가독성을 위해 머리·무기를 강조).
- 머리·몸통·팔·다리·무기가 실루엣으로 명확히 구분. 무기는 몸과 구분되는 실루엣.
- 등급(외형 승급)이 올라가도 같은 얼굴·체형·외곽선 스타일 유지.

**몬스터**
- 머리·몸통·다리·꼬리가 실루엣으로 명확히 구분.
- 종류는 색만이 아니라 **실루엣·체형**으로 구분.
- 작은 일반 적보다 엘리트·보스의 실루엣을 더 크고 무겁게.

## 크기 기준 (현재 코드 프로필 = 표시 기준 박스)

값은 `EnemyProfiles.gd`의 `size`(화면 표시 px 기준). 전신 스프라이트는 이 박스에 맞춰 스케일·정렬하고(발 바닥선 y=690 기준), 실제 그림이 박스보다 크면 비율을 유지한 채 맞춘다. **일반 적 < 엘리트 < 보스**가 실루엣 크기·무게감으로 드러나야 한다.

| 등급 | 대상 | 기준 박스(px) |
|---|---|---|
| 플레이어 | 용병 | 60 × 90 |
| 일반 적 | 고블린 | 42 × 64 |
| 일반 적 | 굶주린 늑대 | 78 × 44 |
| 일반 적 | 방패병 | 66 × 96 |
| 엘리트 | 오우거 징수꾼 | 108 × 132 |
| 보스 | 철퇴의 브루노 | 132 × 156 |

- 박스 크기·위치는 전투 판정(거리·체력바) 기준이므로 임의 변경 금지(변경 필요 시 기획 보고).

## 금지 사례

- 실사풍, 고해상도 사진풍.
- 고프레임 픽셀 아트(캐릭터당 수십 프레임)·스프라이트 시트.
- 과한 질감·장식, 축소 시 뭉개지는 디테일.
- 배경보다 어둡거나 대비가 낮아 묻히는 캐릭터.
- 가로·세로 비율을 다르게 늘린 왜곡.
- **불투명 배경 캐릭터 PNG**(알파 채널 없는 흰/단색 배경).
- 프롬프트만으로 기존 승인 캐릭터 참조 없이 독립 생성.

## 신규 에셋 승인 절차

1. SD 전신 디자인 생성(같은 종족/등급의 승인 에셋·이 문서 참조).
2. 사용자 승인.
3. 투명 배경 PNG로 내보내기(경로·파일명 위 표 기준).
4. Godot Sprite2D 적용·코드 연출 연결 확인(대기·공격·피격·죽음).
5. 아이폰 가독성 확인.

기준 디자인과 결과가 어긋나면 그 단계로 되돌린다. 최종 미술 취향·가독성 승인은 각 아트 TASK의 아이폰 확인에서 한다.
