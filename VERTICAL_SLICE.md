# VERTICAL_SLICE.md — 버티컬 슬라이스 정본 계획

이 문서는 TASK별 작업지시서가 아니라 버티컬 슬라이스 단계 전체의 정본 계획이다. 작업의 정본은 각 GitHub Issue이며, 이 문서는 슬라이스의 범위·방향·순서를 고정한다.

승인: 2026-06-26, GitHub Issue #1(TASK 013) 댓글에서 사용자 승인.

---

## 목표

임시 도형으로 재미를 검증한 프로토타입 v0.1의 첫 지역 흐름을 **그대로 유지**하면서, 실제 출시 게임의 첫 지역처럼 **보이고 들리고 느껴지는 작은 완성품**으로 만든다. 콘텐츠를 늘리는 단계가 아니다.

유지하는 검증된 흐름.

```text
늑대·고블린·방패병 → 오우거 → 철퇴의 브루노 → 빚 500 상환 → 명성 풋내기 용병 → 저장
```

---

## 선택한 아트 방향 — SD 전신 스프라이트

- **화풍·비율·크기·제작 방식·승인 절차의 정본은 `ART_STYLE_GUIDE.md`다**(Issue #3). 이 절은 방향만 요약한다.
- 캐릭터는 **SD 전신 투명 PNG 1장**을 Godot Sprite2D 하나로 적용하고, 대기·전진·공격·피격·죽음을 코드 트윈(이동·확대·회전·플래시)으로 연출한다. 기존 전투 연출이 트윈 기반이라 자연스러운 연장이다. (초기에는 2D 컷아웃 파츠 방식을 택했으나, AI 생성으로 동일 캐릭터의 파츠 비율·관절을 안정적으로 유지하기 어려워 SD 전신으로 변경 — Issue #3.)
- **제작 소스: AI 생성.** 단, 완성 캐릭터 이미지를 나중에 억지로 자르지 않는다. **처음부터 2D 컷아웃 리깅용으로 투명 배경의 분리 파츠 세트**를 만든다.
- 피하는 방식: 고프레임 픽셀, 캐릭터당 수십 프레임, 3D, 실사풍 고해상도, 모바일에서 실루엣이 안 구분되는 세밀한 화풍.

## 첫 지역 분위기 — 황량한 변경 교역로

- 빚에 쫓겨 일거리를 찾아 걷는 변경의 교역로. 가로 자동 전진(길을 걷는 느낌)과 맞고, 굶주린 늑대·고블린·방패병·오우거 징수꾼·불법 용병단 간부 브루노가 모두 "교역로의 위협"으로 묶인다.
- 색·명도: 배경은 탁한 저채도(흙길·메마른 덤불·흐린 하늘). 용병·적·공격 효과·피해 숫자·UI가 항상 배경보다 밝게. 배경이 캐릭터보다 튀지 않게.
- 실루엣: 용병·늑대·고블린·방패병·오우거·브루노는 색을 빼도 형태로 구분돼야 한다.

## 첫 아트 샘플 범위 (먼저 이것만)

- 용병 1명, 굶주린 늑대 1마리, 변경 교역로 배경 1개만 제작한다.
- 용병 최소 동작: 대기·전진·기본 공격·피격 + 강타 또는 반격 중 하나.
- 늑대 최소 동작: 접근·공격·피격·사망.
- 배경: 가로 스크롤·전투/ UI를 가리지 않는 명도·첫 지역 분위기.
- **아이폰에서 이 샘플이 승인된 뒤** 고블린·방패병·오우거·브루노로 확장한다. 첫 샘플에 나머지 적·장비·완성 이펙트 전체는 포함하지 않는다.

## 애니메이션 범위

- 기존 트윈 재사용 최소 세트. 대기(미세 흔들림)·전진(다리 스웨이, 전진감은 배경 스크롤 유지)·공격(무기 든 팔 회전)·피격(기존 flash/tint)·사망(기존 페이드+축소). 강타=큰 스윙+기존 금색, 반격=막기 번쩍+되받음 잔상.
- 상태머신·AnimationPlayer 없이 `create_tween`으로 시작. 자동 전투를 오래 보는 게임이므로 매 공격 큰 화면 흔들림·강섬광은 쓰지 않는다.

## UI 방향 (540×960)

기능은 유지하고 정보 구조만 재배치한다.

- 항상 표시: 레벨·EXP 바·골드·적 HP(적 머리 위).
- 슬림 목표줄: 빚·명성.
- 패널로: 세부 능력치·특성·저장 초기화(현재 특성 패널 유지).
- 보스전 정보(체력·시간·방어 태세·자세 붕괴)는 보스전에서만 상단 강조.
- 하단 강화 버튼 3개는 현재 크기·위치(엄지 도달) 유지, 정확한 픽셀은 실제 화면 보며 미세 조정.

## 사운드 범위

- 필수(9종): 기본 공격·피격·강타·반격(방어/되받음)·레벨업·엘리트 등장·보스 등장·보스 강공격 준비·보스 승리·버튼. (반격은 방어+공격을 한 묶음으로 본다.)
- 있으면 좋음: 연격·자세 붕괴·용병 피격 구분.
- 보류: BGM(또는 저음량 앰비언트 1개만 선택). 라이선스 미확인 에셋 금지, **CC0 우선**.

## 외형 성장 (포함)

- 2단계만. 시작(낡은 검·천옷·초보 자세) / 브루노 처치 후(철검·가죽 갑옷·조금 안정된 자세). 명성 풋내기와 연결.
- 저장: 새 항목을 추가하지 않고 **기존 `boss_defeated`로 외형을 선택**한다(불러오기 시에도 유지).
- 랜덤 아이템·인벤토리가 아니라 고정된 첫 지역 완료 외형 승급이다.

## 오프라인 보상 — 이번 슬라이스 제외

- 슬라이스 목적(완성품 외형·사운드)과 무관하고, 엄격한 저장 검증·성장 밸런스·iOS Safari 시간 처리와 충돌 위험이 있어 제외한다. 슬라이스 승인 후 별도 Issue에서 다시 검토한다.

## 코드 정리 범위 (최소)

- TASK 014에서 **동작 변화 0**으로 다음만 한다. 적·보스 프로필/밸런스 상수를 `game_data.gd`로 분리(preload, 값 불변), 빈 `_play_sfx(name)` 호출 훅 추가(소리는 TASK 019). 유닛 외형 생성 경계는 이미 `_build_merc`/`_build_enemy`/`_spawn_boss`에 있으므로 추가 구조 불필요.

- 대규모 리팩터링·Autoload 전환·테스트 없는 구조 변경 금지. `--verify` TASK_001~011 ALL PASS 유지로 확인.

## 완료 조건 (버티컬 슬라이스)

- 용병·첫 지역 적이 임시 사각형이 아니다(2D 컷아웃).
- 변경 교역로 배경 적용.
- 핵심 전투 동작이 캐릭터 애니메이션으로 보인다.
- 강타·연격·반격이 시각과 소리로 구분된다.
- UI가 모바일 게임 화면처럼 정리된다.
- 엘리트·보스가 시각적으로 구분된다.
- 사망·부활·승리 연출이 이해된다.
- 브루노 처치 후 외형 승급이 보인다.
- 저장과 전체 전투 흐름이 유지된다.
- 아이폰에서 처음부터 보스 완료까지 재검증된다.
- 단순히 에셋 파일이 추가된 것을 완료로 보지 않는다.
