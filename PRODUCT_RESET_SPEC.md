# PRODUCT_RESET_SPEC.md — RESET 001

> 이 파일은 RESET 작업의 **정본**이다. 판단이 흔들리면 이 문서를 우선 기준으로 삼는다.
> 작업 방식: 딸깍 모드. 세부를 묻지 않고 Claude가 판단해 한 번에 만든다.

## 목표

백지에서 빠르게, 작지만 **출시 가능한 구조**를 가진 세로형 모바일 방치 RPG **v0.1 후보**를 만든다.
프로토타입 데모가 아니라 출시 후보다. 처음 10초에 모바일 게임처럼 보여야 한다.

## 만든 게임 (Claude 결정)

- **제목**: 균열기사 (Riftblade Knight)
- **컨셉**: 갈라진 변경의 균열에서 끝없이 밀려오는 마물을 홀로 베어내는 기사. 자동 전투로 전진하며 강해진다.
- **지역**: 갈라진 변경 — 20스테이지, 5스테이지마다 보스, 20층 최종 보스. 이후는 무한 스케일(엔드리스)로 루프 유지.
- **재화**: 골드.
- **성장 5종**: 공격력 · 체력 · 방어력 · 치명타 · 골드 획득.

## v0.1 범위 (이번에 구현)

1 지역 / 20 스테이지 / 일반 적 외형 다수 + 보스 외형 / 성장 버튼 5 / 자동 전투 / 레벨업 / 골드 / 스테이지 진행 / 보스전(시각 구분) / 클리어·보상 표시 / 저장·불러오기 / 설정(진행 초기화) / Web 빌드 / 540×960.

## 구조 원칙 (출시 가능 구조)

- **데이터/화면 분리**: 모든 밸런스·콘텐츠 수치는 `reset/GameData.gd` 한 곳. 화면·루프는 `Main.gd`.
- 저장은 `reset/SaveSystem.gd`(JSON, 버전 필드). 즉흥적이지 않게 키 고정.
- 적/스테이지/성장 수치는 공식·표로 두어 나중에 쉽게 변경.
- 아트 에셋·사운드는 나중에 교체하기 쉽게 경계만(지금은 도형).
- 과한 아키텍처 금지(매니저 남발·플러그인·상태머신 대공사 없음).

## 아트 방침

도형 기반 임시 아트. **AI 생성·투명 PNG 제작·무료 에셋 조사 안 함.** 단, 패널·숫자·피해·보상 표시를 조합해 테스트 화면이 아니라 모바일 게임처럼 보이게 한다. 색은 어두운 판타지 + 골드 강조.

## 기존 코드 처리 (Claude 결정)

기존 `Battle.gd`(PNG 아트 의존·TASK 검증 하네스·옛 세계관)는 **재사용하지 않는다.** 백지 전제에 맞춰 새 씬·새 스크립트로 작성하고 메인 씬을 교체한다. 기존 파일은 삭제하지 않고 남겨 둔다(비파괴). Godot 프로젝트 설정·폰트·Web 프리셋만 재사용.

## 하지 않을 것 (이번 1차)

장비/인벤토리/수집/스킬트리/퀘스트/다지역 대량/정식 아트/정식 사운드/광고·과금/랭킹/로그인/서버/긴 문서/과한 테스트 하네스.

## 완료·자체 검수 기준

실행·Web 빌드 성공 / 540×960 / 첫 화면이 게임처럼 보임 / 자동 전투·성장·스테이지·보스·보상·저장·재접속 유지 / 아이폰 확인 가능. 처음 10초 게임처럼 보이는가·3분 할 일 있는가·강해지는 체감·보스까지 흐름·재접속 유지·UI 안 깨짐·허전하지 않은가.
