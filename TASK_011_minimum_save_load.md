# TASK 011 — 최소 저장·불러오기와 임시 특성 포인트 제거

## 작업 목적

핵심 성장 상태를 기기에 저장하고, 다시 실행했을 때 자동 복원한다.

흐름: 게임 진행 → 골드·경험치·강화·특성 변경 → 자동 저장 → 새로고침/종료 → 자동 불러오기 → 이전 성장 상태로 전투 재개.

함께 처리할 항목.

- DEBUG_START_TRAIT_POINTS = 3 임시 기능 제거. 정상 게임은 레벨 10마다 얻는 포인트만 사용.
- 저장 파일 초기화 기능 추가.
- 손상·미지원 저장 파일이 있어도 게임이 중단되지 않게 처리.

핵심 질문: 아이폰에서 닫거나 새로고침한 뒤에도 성장과 보스 진행이 안정적으로 유지되는가. 오프라인 보상은 구현하지 않는다.

## 먼저 읽을 문서

CLAUDE.md → UNKNOWN.md → HANDOFF.md → TASK_009 → TASK_010 → checklist.md → context-notes.md → 현재 Battle.gd. 기존 TASK 001~010 기능·자동 검증을 유지하고 전면 재작성하지 않는다.

## 문서 상태 동기화

사용자가 아이폰에서 TASK 010까지 확인 완료. HANDOFF.md·checklist.md에 "TASK 010 아이폰 확인 완료(빚·명성·결과 화면)"를 반영하고 대기 상태로 남기지 않는다. TASK 010 수치·결과 화면은 변경하지 않는다.

## 저장 방식

- 저장 경로: `user://save_v1.json`. 검증 경로: `user://save_test_v1.json`(검증이 실제 저장을 읽거나 덮어쓰지 않는다). 경로 반환 함수로 분기.
- JSON 단일 파일. 최상단 `version`. `const SAVE_VERSION := 1`. Node/객체 저장 금지, 숫자·문자열·불리언·단순 Dictionary만.

## 저장할 항목

- 기본 성장: 레벨, 경험치, 골드, 공격/체력/방어 강화 횟수. (능력치·강화 비용은 저장하지 않고 불러올 때 재계산)
- 전투 특성: 강타/연격/반격 투자 단계(각 0 또는 1). 총/보유 포인트는 레벨과 투자로 재계산. 강타·반격 카운터는 저장하지 않고 0으로 시작.
- 진행: 브루노 처치 여부, 정산 완료 여부.
- 빚·명성: 남은 빚, 누적 상환액, 명성 등급, 명성 상승 여부.

전투 위치는 저장하지 않는다. 불러오면 항상 첫 늑대부터 시작.

## 저장하지 않을 런타임 상태

현재 체력/적 체력/적 종류/순환 위치/고블린 대기열/오우거 순서/보스 체력/제한 시간/방어 태세/자세 붕괴/도전 중 여부/강타·반격 카운터/피해 숫자/패널 상태/처치 수/알림·애니메이션. 불러오기 후: 용병 완전 회복, 첫 늑대 시작, 카운터 0, 모든 팝업 닫힘. 브루노 처치 여부와 성장만 유지.

## 저장 구조(권장)

```json
{ "version": 1,
  "player": { "level": .., "exp": .., "gold": .. },
  "upgrades": { "attack": .., "hp": .., "defense": .. },
  "traits": { "power": .., "combo": .., "counter": .. },
  "progress": { "boss_defeated": .., "bruno_settlement_done": ..,
                "debt_remaining": .., "debt_paid_total": ..,
                "fame_rank": .., "fame_advanced": .. } }
```

## 임시 특성 포인트 제거

`DEBUG_START_TRAIT_POINTS` 상수와 _ready()의 3포인트 지급 코드를 제거한다. 신규 게임은 레벨 1·포인트 0·특성 0으로 시작하고, 레벨 10에 1포인트를 얻는다. 검증은 필요한 값을 직접 설정한다.

## 불러오기 순서·재계산

_ready(): 배경·UI 생성 → 용병 생성 → 저장 불러오기 → UI 갱신 → 첫 적 생성. 저장이 없으면 초기 상태.

재계산.
- 공격력 = MERC_ATK + 공격강화×2 + (level-START_LEVEL)×1.
- 최대 체력 = MERC_MAX_HP + 체력강화×20 + (level-START_LEVEL)×5. 불러오기 후 현재 체력 = 최대 체력.
- 방어력 = MERC_DEF + 방어강화×1.
- 강화 비용: `_cost_after_upgrades(base, mult, count)`로 반복 반올림.
- 총 포인트 = level/10, 보유 = 총 - (강타+연격+반격). 음수면 오류.
- 경험치 정규화: 필요량 이상이면 레벨업 규칙으로 정규화(연출·패널 자동 열기 없이).

## 저장 검증·일관성

JSON 파싱 성공, 최상단 Dictionary, version==1, player/upgrades/traits/progress Dictionary, level≥1, exp≥0, gold≥0, 강화≥0, 특성 0/1, 투자합≤획득, 빚 0~10000, 누적 0~10000, 명성 ∈ {무명 용병, 풋내기 용병}. 숫자는 명시적 int 변환.

정상 진행 상태 2종만 인정.
- 미처치: boss_defeated=false, 정산 false, 무명, 빚 10,000, 누적 0.
- 처치: boss_defeated=true, 정산 true, 풋내기, 빚 9,500, 누적 500.
그 외(미처치인데 풋내기 등)는 오류 → 새 게임.

## 저장·자동 저장

- `_save_game()`: FileAccess WRITE, 실패해도 중단 없이 로그·짧은 안내. 성공 시 JSON.stringify→store_string→flush→close. 암호화·압축 없음.
- 자동 저장: `save_dirty`/`autosave_timer`, `AUTOSAVE_DELAY := 2.0`. 변경 시 dirty=true, 첫 변경에서 타이머 시작(추가 변경에 리셋하지 않아 무한 미룸 방지).
- 지연 저장 대상: 적 처치 골드/경험치, 레벨업.
- 즉시 저장 대상: 강화 버튼, 특성 투자/초기화, 브루노 처치 정산(결과 패널 표시 전 저장 완료).

## 불러오기 안내·Web

- 정상: "저장 데이터 불러옴" 약 0.8~1초, 전투 막지 않음.
- 없음: 안내 없이 새 게임.
- 손상/미지원: "저장 파일을 불러올 수 없습니다. 새 게임으로 시작합니다." 후 새 게임(부분 적용 금지).
- Web에서 영구 저장 불가로 판단되면 "브라우저 설정에 따라 저장이 유지되지 않을 수 있습니다. 비공개 모드는 사용하지 마세요." 경고(중단 없음).

## 저장 초기화

특성 패널 아래에 "저장 초기화" 버튼 추가(특성만 초기화하는 무료 초기화와 구분). 2회 확인: 첫 누름은 약 3초간 "다시 누르면 전체 진행 초기화"로 바뀌고, 두 번째 누름에서 저장 파일 삭제 후 씬 재시작. 3초 지나면 원래 버튼. 복잡한 팝업은 없음. 초기화 후 완전 신규 상태(레벨 1·골드 0·강화 0·포인트 0·특성 0·브루노 미처치·빚 10,000·누적 0·무명). 임시 3포인트 재지급 없음.

## UI 갱신·전투 재시작

불러오기 후 레벨·경험치·바·골드·능력치·강화 버튼·특성·포인트·빚·명성·보스 진행·첫 지역 완료 표시를 모두 갱신(초기값 잔류 금지). 런타임 초기화: 현재 체력=최대, WALK, 첫 적 늑대, enemy_seq_index=0, boss_due=false, boss_fight_active=false, 보스 타이머 기본, 카운터 0, 고블린 대기열 비움, 팝업 닫힘. boss_defeated는 저장값 유지.

## 자동 검증 종료 조건

1. 기존 TASK 001~010. 2. 저장 없음 신규 게임. 3. 저장·불러오기 왕복. 4. 파생 능력치. 5. 강화 비용. 6. 특성 포인트. 7. 브루노·빚·명성 복원. 8. 런타임 전투 상태 미복원. 9. 손상 JSON 안전. 10. 미지원 버전 안전. 11. 잘못된 특성 거부. 12. 저장 초기화. 13. 실제 저장 경로 비접근. 14. 테스트 저장 파일 정리.

검증 상태 예: 레벨 12·경험치 7·골드 123·강화 2/1/3·특성 1/0/0·브루노 처치·빚 9,500·누적 500·풋내기·정산 완료 → 저장 → 초기화 → 불러오기 → 정확 복원. 파생: 공격 25·최대체력 175·방어 5, 현재 체력=최대. 비용: 공격 39·체력 28·방어 69. 포인트: 총 1·강타 1·보유 0.

최종 로그 예: `[VERIFY] ALL PASS TASK_001~011`. 실패 시 예상/실제 출력 후 종료 코드 1.
