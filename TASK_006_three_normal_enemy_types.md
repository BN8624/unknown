# TASK 006 — 일반 적 3종과 전투 순환 구현

## 작업 목적

임시 적 한 종류를 굶주린 늑대·고블린 무리·방패병 세 종류로 교체한다. 각 적은 체력·공격력·방어력·공격 속도·등장 방식이 다르다.

```text
늑대와 기본 전투
→ 고블린 무리 연속 등장
→ 방패병과 장기 전투
→ 다시 늑대로 순환
```

핵심 질문.

> 같은 자동전투라도 적의 성격에 따라 전투 양상이 달라지고, 강타·연격·반격을 바꿔볼 이유가 생기는가.

이번 작업은 일반 적만. 엘리트·보스는 구현하지 않는다.

## 유지할 기능 / 템포

자동 전진·전투·반복, 골드·경험치, 공격력·체력·방어력 강화, 레벨업·완전 회복, 레벨 10 특성, 강타·연격·반격 1단계, 무료 초기화, 피해 숫자, 사망 재시작, TASK 001~005 검증, Web·한글 폰트. TASK 005에서 만족한 템포 유지(용병 0.7초, 빠른 접근·짧은 재등장). 전투를 다시 느리게 만들지 않는다.

## 구현 구조

### 1. 적 프로필 (한 곳에서 관리)

```text
wolf  : 굶주린 늑대  hp32 atk5 def0 interval0.75 approach230 gold5 exp5
goblin: 고블린       hp22 atk4 def0 interval1.0  approach200 gold4 exp4
shield: 방패병       hp75 atk8 def5 interval1.5  approach130 gold12 exp10
```

프로필 원본은 직접 변경하지 않고, 생성 시 필요한 값을 복사해 현재 enemy에 넣는다. 별도 JSON·DB·편집기는 만들지 않는다.

### 2. 적 역할

- 늑대: 빠른 접근·잦은 공격, 낮은 체력·방어. 기본 비교 기준.
- 고블린: 약하지만 3마리가 짧은 간격으로 연속 등장(동시 다중 전투 아님). 무리 내부 재등장 0.12~0.2초, 무리 끝→방패병은 일반 0.4초.
- 방패병: 높은 체력·방어로 한 전투를 오래 끔. 강타가 상대적으로 유리하되 강타 전용은 아님. 어떤 특성으로도 처치 가능.

### 3. 결정적 순환 (무작위 없음)

```text
["wolf","wolf","wolf","goblin","goblin","goblin","shield"]  반복
```

처치 수/순환 인덱스로 다음 적 결정. 같은 조건이면 항상 같은 순서. 일반 플레이도 이번 단계는 무작위 없음.

### 4. 첫 시작 / 재시작

게임 시작 첫 적은 항상 늑대. 용병 사망 후 재시작은 현재 순환 위치 유지(처음으로 초기화 안 함). Web 새로 열거나 완전 재시작 시 첫 늑대부터. 저장은 아직 없음.

### 5~6. 보상 / 성장 속도

처치 시 현재 적 프로필의 골드·경험치 지급(늑대 5/5, 고블린 4/4, 방패병 12/10). 반격 처치도 정확히 한 번. 레벨업 필요 경험치·강화 비용·시작 능력치·특성 배율·템포는 변경하지 않는다.

### 7. 임시 도형 (크기·실루엣 구분)

- 늑대: 낮고 가로로 긴 몸체, 붉은/갈색, 빠른 접근.
- 고블린: 작고 세로로 선 좁고 키 큰 몸체, 녹색.
- 방패병: 가장 크고 무거운 회색/청회색, 앞에 작은 방패 도형(자식 노드, 시각용), 느린 접근.

복잡한 스프라이트·애니메이션 없음.

### 8. 적 이름 표시

적 체력 바 위에 이름 표시(예 "굶주린 늑대", "고블린 무리 2/3", "방패병"). 고블린은 무리 번호 표시. 라벨은 적과 함께 이동, 사망 시 제거. 한글 폰트 적용. 상단 상태가 너무 길어지지 않게.

### 9. 체력 바

최대 체력이 달라도 바 길이는 동일, 비율만 반영(현재÷최대). 새 적 등장 시 이전 적 바·이름표가 남지 않음. 방패병 체력은 감소 속도로 체감.

---

## 특성과 적

- 강타×방패병: 방어력 5라 배율 높은 강타가 상대적으로 유리(`round(atk×2.2)−def`). 관통·보너스 없음. 강타 없이도 처치 가능.
- 연격×고블린: 추가 베기로 마무리, 무리 처리 시간 단축. 첫 공격으로 죽으면 추가 베기 없음. 처치 후 돌진은 아직 없음.
- 반격: 늑대의 잦은 공격으로 발동을 자주 봄. 본격 강공격 대응 검증은 다음 엘리트 작업으로. 반격 수치·확률 변경 없음.

---

## 적 Dictionary / 정리

enemy에 type, name, max_hp, hp, atk, defense, interval, approach_speed, gold_reward, exp_reward, state와 기존 몸체·체력 바·타이머 필드 포함. 이동은 전역이 아니라 현재 적 approach_speed, 공격 간격은 현재 적 interval 사용. 적/용병 사망으로 제거 시 몸체·체력 바 배경·체력 바·이름 라벨·방패 도형을 모두 정리(잔존·겹침 금지).

고블린 무리 번호(1/3, 2/3, 3/3)는 순환 배열 위치나 별도 인덱스로 추적. 3번째 죽으면 일반 지연 후 방패병으로.

## 자동 검증

기존 TASK 001~005 검증을 삭제·완화하지 않는다. 적 순환으로 기존 보상 기대가 어긋나지 않도록 **검증 모드의 기존 성장 루프에서는 기준 적(보상 5/5)만 생성**한다(더 단순한 방식 선택). TASK 006 검증은 별도로 결정적으로 수행.

- 프로필 수치: 위 표와 일치(불일치 시 종류·예상/실제 로그).
- 등장 순서: 첫 14회가 늑대3·고블린3·방패병1 ×2와 일치(무작위 비의존). 고블린 무리 번호 1/3·2/3·3/3.
- 보상: 늑대 5/5, 고블린 4/4, 방패병 12/10. 반격 경로도 한 번만.
- 주는 피해: 용병 공격력 10 → 늑대/고블린(def0) 10, 방패병(def5) 5. 강타(atk10, 방패병 def5) `round(22)−5=17`. 적이 주는 피해(용병 def2): 늑대 3, 고블린 2, 방패병 6. 최소 1 유지.

종료 조건: 기존 TASK 001~005 + 프로필 + 순서 + 무리 번호 + 보상 + 주는 피해 + 방패병 일반/강타 비교 + 불변식 모두 통과 시 종료 0. 통과 안 했는데 ALL PASS만 찍고 끝내지 않음. 최종 로그에 TASK_006 포함 표시.

## 검증 / 재미 판단

PC 화면(가능 시): 세 적 모양·크기 구분, 이름표·체력 바 겹침 없음, 고블린 3연속, 무리 후 방패병 전환, 방패병 전투가 길지만 답답하지 않음, 특성 패널 열고 닫아도 순환 유지, 적 교체 시 잔존 없음. 못 하면 미검증 기록.
