# HANDOFF

## 현재 목표

강타·연격·반격이 단순 능력치 증가가 아니라 서로 다른 플레이 스타일로 보이고 느껴지는지 확인한다. (TASK_005)

## 지금까지 완료

- TASK_001: 자동전투 루프, 임시 도형·체력 바·타격 표현·용병 사망 안전장치.
- 아이폰 테스트용 웹(HTML5) 내보내기 + Tailscale HTTPS 노출, 한글 폰트 적용(폰트 비공개).
- TASK_002: 골드 +5, 공격력 강화. 아이폰 확인됨.
- TASK_003: 경험치 +5, 레벨/경험치 바, 레벨업·연출. 아이폰 확인됨.
- TASK_004: 체력·방어력 강화, 방어력 피해 계산(최소 1), 적 공격력 4→6 상쇄, 강화 버튼 3개, 피해 숫자. 아이폰 확인됨(선택 재미 검증됨).
- TASK_005: 레벨 10마다 특성 포인트, 특성 패널(투자·무료 초기화), 강타(4타째 220%)·연격(30% 60% 추가 베기)·반격(25% 40%↓ + 120%), 충돌 우선순위, 특성별 전투 문구.

## 특성 작동 요약

- 강타: 기본 공격 4번째마다 round(공격력×2.2)−방어력 피해. 큰 전진·적 밀림·큰 금색 문구.
- 연격: 일반 기본 공격 시 30% 확률로 round(공격력×0.6)−방어력 추가 베기 1회(재발동 없음, 강타 공격엔 미발동).
- 반격: 피격 시 25% 확률로 받는 피해 ×0.6(40% 감소), 생존 시 round(공격력×1.2)−방어력로 즉시 반격.
- 일반 플레이는 확률, 헤드리스 검증은 force 플래그 강제 경로.

## 실행 방법

방법 A — 에디터로 실행 (권장, 화면 확인용).

```
"C:\Users\USER\godot-engine\Godot_v4.7-stable_win64.exe" --path "C:\Users\USER\unknown"
```

에디터가 열리면 F5(실행)로 전투 화면을 본다. 또는 다음으로 바로 실행한다.

```
"C:\Users\USER\godot-engine\Godot_v4.7-stable_win64.exe" --path "C:\Users\USER\unknown" Battle.tscn
```

방법 B — 자동 검증 (화면 없이 로그만).

```
"C:\Users\USER\godot-engine\Godot_v4.7-stable_win64_console.exe" --headless --path "C:\Users\USER\unknown" -- --verify
```

## 검증 상태

- 헤드리스 자동 검증(--verify): 기존 TASK_001~004 + 특성 지급(레벨10 +1·중복 없음)·투자/무료 초기화·강타(4타째, atk17 def3 → 34)·연격(round(17×0.6)=10)·반격(받는 4→2, 반격 20)·충돌(4타 강타만) **ALL PASS**, 종료 코드 0, 경고/오류 0건. 확률 비의존(강제 경로).
- 라이브(비검증) 기본 루프 5초 스모크: 런타임 오류 0.
- PC 화면 검증: 미실행(헤드리스만). 아이폰 검증: 대기.

## 남은 위험

- 특성의 실제 전투 연출(강타 묵직함, 연격 빠름·연속, 반격 방어→되받음)과 패널 가독성·조작감은 아이폰에서 직접 비교 필요(미검증).
- 라이브에서 특성을 투자한 상태의 전투 경로는 공식·결정 함수로 검증했으나 화면 동작 자체는 미확인(자동 검증은 강제 경로 단위 검증). 첫 특성 사용 시 이상하면 알려주면 조정.
- 일반 플레이로 레벨 10까지는 방치로 다소 시간이 걸림. 특성을 빨리 보려면 강화로 처치 속도를 올리면 됨. 성장 속도 조정은 `Battle.gd` 상단 상수에서 가능.
- 사망/재시작 연출이 약한 점은 그대로 보류.
