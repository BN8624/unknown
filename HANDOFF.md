## 입력 버그 수정 요약 (아이폰 피드백, 확인 완료)

- 증상: 보스 도전 중 강화 버튼이 안 눌림 / 특성 투자 후 닫기가 여러 번 눌러야 닫힘 / 오우거 진입 전 특성창 열면 처치해야 닫힘.
- 원인: Godot의 Control 입력 픽킹은 z_index가 아니라 트리 순서(나중 형제가 위)를 따른다. 전투 중 동적 생성되는 표시 노드(떠오르는 피해 숫자 Label, 잔상 ColorRect, 적·보스 도형과 체력 바)와 전체 화면 보스 정보 패널이 기본 `mouse_filter=STOP`이라, 패널·버튼 위에서 터치를 가로챘다.
- 해결: 비상호작용 표시물 전부 `MOUSE_FILTER_IGNORE`(`_ignore_mouse`/`_ignore_enemy_mouse`/`_ignore_decorative_mouse`). 버튼과 특성 패널 배경만 STOP. 원칙: 표시 노드는 입력 통과, 실제 버튼/모달만 입력을 잡는다.

- TASK_009 검증 항목: 보스 프로필 / 오우거 다음 등장·처치 후 재등장 방지 / 기본·기본·강공격 패턴 / 기본 13·강공격 33 / 반격 3번째 연동(33→20) / 방어력 6·9·4 / 자세 붕괴 8타·미달 초기화·붕괴 중 미연장 / 강타 보스 보너스(18 > 일반 16).
- TASK_010 검증 항목: 초기 빚 10,000·무명·정산 미완료 / 패배 복귀에 빚·명성·정산 불변 / 정산(골드 100→300·빚 10,000→9,500·무명→풋내기) / 골드 비차감 / 중복 정산 방지 / 빚 0 하한(300+500상환→실제 300·남은 0) / UI 텍스트 갱신.
- TASK_011 검증 항목: 신규 게임(임시 포인트 없음) / 저장·불러오기 왕복 / 파생 능력치(25·175·5) / 강화 비용(39·28·69) / 특성 포인트(총1·보유0) / 런타임 전투 상태 미복원(boss_defeated만 유지) / 손상 JSON·미지원 버전·잘못된 특성 폴백 / 저장 초기화 / 실제 저장 경로 비접근·테스트 파일 정리.
- 라이브 스모크(헤드리스 플레이): TASK_009 보스 전투/재도전·TASK_010 정산·결과 패널 런타임 오류 0. TASK_011 실제 플레이 자동 저장→재실행 불러오기(lv2·골드39·atk11) 런타임 오류 0, 실제 저장 경로만 사용(검증은 테스트 경로만).
- 아이폰 플레이: **TASK_001~011 전부 확인됨**(저장 지속성·입력 수정 포함, 잘 됨).

## 남은 위험

- iOS Safari의 IndexedDB 지속성(저장 용량·기간 제한, 캐시 정리, 비공개 모드 영구 저장 불가)에 따라 드물게 저장이 사라질 수 있음 — 장시간/여러 날 플레이 시 재확인 가치 있음.
- 보스 난이도·45초 제한이 장기 플레이에서 적당한지(너무 조급/답답하지 않은지)는 전체 흐름 점검 때 재확인. 답답하면 체력·제한 시간·접근 속도·방어 태세 지속·자세 붕괴 타격 수를 조정.
- 저장 없음(또는 초기화) 상태에서 시작하면 빚 10,000·무명·보스 미처치. 저장 파일이 있으면 복원. 설계대로.
