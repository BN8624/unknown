## 다음 액션

> 사용자가 아이폰에서 경험치·레벨업 성장 체감을 확인한 뒤, 현재 루프를 수정하거나 체력·방어력 강화 또는 첫 특성 시스템 중 다음 작업을 결정한다.

## 실행 방법

방법 A — 에디터로 실행 (권장, 화면 확인용).

```
"C:\Users\USER\godot-engine\Godot_v4.7-stable_win64.exe" --path "C:\Users\USER\unknown"
```

에디터가 열리면 F5(실행)로 전투 화면을 본다. 또는 다음으로 바로 실행한다.

```
"C:\Users\USER\godot-engine\Godot_v4.7-stable_win64.exe" --path "C:\Users\USER\unknown" Battle.tscn
```

방법 B — 자동 검증 (화면 없이 로그만).

```
"C:\Users\USER\godot-engine\Godot_v4.7-stable_win64_console.exe" --headless --path "C:\Users\USER\unknown" -- --verify
```

## 검증 상태

- 헤드리스 자동 검증(--verify): 경험치 처치당 +5, kill5에서 레벨2(exp 0/30, 공격력 직전+1=13, 최대 체력 +5=105 완전 회복) task003 PASS, kill11에서 골드 강화 task002 PASS, **ALL PASS**, 종료 코드 0, 경고/오류 0건.
- PC 화면 검증: 미실행(헤드리스만). 아이폰 검증: 대기.

## 남은 위험

- 경험치 바·레벨업 문구·용병 강조의 실제 가독성과 성장 체감은 아이폰 플레이로 확인 필요(미검증).
- 레벨업이 매번 완전 회복을 주므로, 기존 "전투 사이 미회복으로 약 12처치마다 사망" 문제는 레벨이 오르는 동안에는 완화됨. 사망/재시작 연출이 약한 점은 그대로 보류.
- 첫 레벨업까지 5처치, 첫 체감 공격력 차이는 16부터. 성장 속도가 적절한지는 추가 플레이로 판단. 수치는 `Battle.gd` 상단 상수에서 조정 가능.
