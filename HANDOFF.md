## TASK_009 구현 요약

- 브루노 프로필: 체력 850 · 공격력 15 · 기본 방어력 6 · 보상 골드 200/경험치 100 · 공격 간격 2.0초 · 제한 시간 45초 · 강공격 배율 2.3.
- 등장: 오우거 처치 시 `boss_due`로 전환, 다음 스폰에서 일반 적 대신 브루노 생성. 짙은 자주색 큰 몸체·큰 철퇴(손잡이+철구 Node2D)·넓은 체력 바·[보스] 이름표. 상단 보스 전용 UI(이름·HP·남은 시간·현재 상태).
- 공격 패턴: 기본·기본·강한 내려찍기 반복. 강공격은 1.2초 준비 동작(철퇴 들기·주황 강조·"강한 내려찍기 준비!", 준비 중 무피해). 준비 중 사망·자세 붕괴 시 취소.
- 피해: 기본 15−방어(=13), 강공격 round(15×2.3)−방어(=33). round(15×2.3)는 부동소수 오차를 피해 ×23÷10로 계산해 35가 되게 함. 반격이 3번째 강공격에 맞물려 33→20으로 감소 후 되받음.
- 방어 태세: 전투 시작 10초마다 3초, 방어력 6→9(round(6×1.5)). 강공격 준비·자세 붕괴 중에는 시작을 미루고, 자세 붕괴 발생 시 즉시 해제.
- 자세 붕괴(연격의 역할): 4초 안에 8타(일반·연격 추가 베기·강타·반격 각 1타)를 맞으면 3초간 무방비, 방어력 6→4(round(6×0.6)), 공격·강공격 준비 중단. 4초 내 8타 미달이면 타격 수 0으로 초기화, 붕괴 중에는 재누적·연장 없음.
- 강타 보스 추가 피해 10%: 브루노에게만 round(공격력×2.2×1.1)−현재 방어력. 일반 적·오우거엔 기존 강타 공식 유지(TASK_009에서 최초 적용).
- 현재 방어력: 자세 붕괴 4 > 방어 태세 9 > 일반 6 우선순위를 보조 함수 하나(`_enemy_def`)로 통일. 일반·강타·연격·반격 모두 같은 값 사용.
- 제한 시간: 실제 전투 진입 순간부터 45초 감소(접근·등장 연출 중에는 정지). 0이면 화력 부족.
- 패배 원인: 용병 체력 0 → 생존 부족, 시간 초과 → 화력 부족. 약 2초 안내 후 자동으로 첫 늑대 복귀(보상 없음).
- 자동 사냥 복귀·재도전: 패배 시 보스 정리·첫 늑대 복귀, 일반 적과 오우거를 다시 거쳐 재도전. 성장·특성·포인트 유지, 브루노 체력은 850으로 초기화.
- 승리: 골드 200·경험치 100 한 번(반격 마무리 포함). "보스 처치! … 첫 지역 클리어!" 표시. `boss_defeated=true`로 같은 실행에서 재등장 차단. 약 2초 후 첫 늑대로 복귀해 일반 사냥 지속.
- 진행 표시: 일반 전투 중 "보스까지 N/8" → 오우거 처치 후 "보스 등장 예정" → "보스 전투" → 처치 후 "첫 지역 완료".


보스까지: 늑대3→고블린3→방패병→오우거를 처치하면 브루노가 등장한다. 시작 특성 포인트(임시) 3개로 강타·연격·반격을 바로 시험할 수 있다.

방법 B — 자동 검증 (화면 없이 로그만).

```
"C:\Users\USER\godot-engine\Godot_v4.7-stable_win64_console.exe" --headless --path "C:\Users\USER\unknown" -- --verify
```

방법 C — 아이폰(웹) 확인.

아이폰 주소 (Tailscale 켠 상태에서 Safari로 연다):

```
https://node.tail3e9e21.ts.net:8443
```

- 같은 Tailscale 망의 아이폰(`iphone182`)에서 접속한다.
- 노출 경로: `tailscale serve`가 `https://node.tail3e9e21.ts.net:8443` → `http://127.0.0.1:8060`로 프록시하고, 그 8060을 아래 정적 서버가 `build/`로 제공한다.
- 같은 파일명이라 Safari가 이전 빌드를 캐시할 수 있다. 보스가 안 보이면 새로고침하거나 주소 끝에 `/?v=숫자`를 붙여 강제로 새로 받는다.

코드 변경 후 아이폰 반영 순서 (웹 재내보내기 → 서버는 디스크에서 바로 제공).

```
# 1) 웹 빌드 재내보내기 (preset 이름은 "Web")
"C:\Users\USER\godot-engine\Godot_v4.7-stable_win64_console.exe" --headless --path "C:\Users\USER\unknown" --export-release "Web" "C:\Users\USER\unknown\build\index.html"

# 2) 정적 서버 (이미 떠 있으면 생략 — build/를 8060으로 제공)
python -m http.server 8060 --bind 127.0.0.1 --directory "C:\Users\USER\unknown\build"
```
