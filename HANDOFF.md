## 실행 방법

방법 A — 에디터로 실행 (권장, 화면 확인용).

```
"C:\Users\USER\godot-engine\Godot_v4.7-stable_win64.exe" --path "C:\Users\USER\unknown"
```

에디터가 열리면 F5(실행)로 전투 화면을 본다. 또는 다음으로 바로 실행한다.

```
"C:\Users\USER\godot-engine\Godot_v4.7-stable_win64.exe" --path "C:\Users\USER\unknown" Battle.tscn
```

방법 B — 자동 검증 (화면 없이 로그만).

```
"C:\Users\USER\godot-engine\Godot_v4.7-stable_win64_console.exe" --headless --path "C:\Users\USER\unknown" -- --verify
```

## 검증 상태

- 헤드리스 자동 검증(--verify): 골드 처치당 +5, 20골드에서 강화(공격력 10→12, 비용 ×1.4), 공격력 16에서 처치 타격 3→2로 감소 PASS, 종료 코드 0, 경고/오류 0건.
- 아이폰 플레이 확인: 한글 표시 정상, 골드 강화 성장 루프 잘 작동.

## 남은 위험

- 용병 사망 후 다시 시작하는 순간이 화면에서 또렷하게 구분되지 않음(현재 연출은 본체 약 1초 회색뿐). 개선 시 사망/재시작 표시 강화 필요(`context-notes.md`).
- 용병 HP가 전투 사이 회복되지 않아 약 12처치마다 사망→전체 회복·리셋된다. 루프는 안 깨지지만 성장 체감을 해칠 수 있음. 회복 추가는 범위 밖이라 보류.
- 강화 버튼/수치 균형(처음 강화까지 4처치, 체감 차이는 공격력 16부터)이 적절한지는 추가 플레이로 확인. 수치는 `Battle.gd` 상단 상수에서 조정 가능.
