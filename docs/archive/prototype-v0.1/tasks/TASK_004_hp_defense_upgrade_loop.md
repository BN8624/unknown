# TASK 004 — 체력·방어력 강화 성장 루프 구현

## 작업 목적

기존 공격력 강화에 이어 최대 체력과 방어력 강화를 추가한다.

```text
적 처치
→ 골드 획득
→ 공격력·체력·방어력 중 하나 선택
→ 선택한 능력치가 실제 전투에 반영
→ 같은 골드를 서로 다른 성장 방향에 사용할 수 있음
```

TASK 004에서 확인할 것.

> 같은 골드를 공격력, 체력, 방어력 중 어디에 사용할지 선택하는 것이 의미 있는 성장 결정으로 느껴지는가.

## 구현 범위

### 1. 최대 체력 강화

강화 1회당 최대 체력 +20, 첫 비용 20골드, 다음 비용 ×1.4.

```text
const HP_UPGRADE_AMOUNT := 20
const HP_UPGRADE_BASE_COST := 20
const HP_COST_MULTIPLIER := 1.4
```

### 2. 체력 강화 시 현재 체력

최대 체력 +20과 함께 현재 체력도 +20(잃은 체력은 유지, 완전 회복 아님). 현재 체력은 새 최대 체력을 초과하지 않는다. 체력 바는 즉시 갱신.

### 3. 방어력 능력치

용병/적 상태에 방어력 필드 추가. 용병 시작 방어력 2, 일반 적 0.

```text
const MERC_DEF := 2
const ENEMY_DEF := 0
```

범용 전투 엔진이나 능력치 클래스는 만들지 않는다.

### 4. 피해 계산 변경

```text
최종 피해 = 공격력 - 대상 방어력
최소 피해 = 1
```

작은 계산 함수 하나로 처리. 강타·연격·반격·관통은 구현하지 않는다.

### 5. 기존 전투 속도 보존

방어력 2 도입에 맞춰 일반 적 공격력을 4 → 6으로 올려 실제 피해 6 - 2 = 4를 유지한다. 난이도 상승이 아니라 상쇄 조정. 용병 시작 공격력·최대 체력은 변경하지 않는다.

### 6. 방어력 강화

강화 1회당 +1, 첫 비용 25골드, 다음 ×1.4.

```text
const DEF_UPGRADE_AMOUNT := 1
const DEF_UPGRADE_BASE_COST := 25
const DEF_COST_MULTIPLIER := 1.4
```

피해는 방어력이 공격력 이상이어도 0이 아니라 최소 1.

### 7. 강화 비용 독립 관리

공격력·체력·방어력의 현재 비용과 강화 횟수를 각각 따로 둔다(`attack_upgrade_count`, `hp_upgrade_count`, `def_upgrade_count`). 한 능력치를 강화해도 다른 능력치 첫 비용은 그대로.

### 8. 강화 UI

버튼 하나를 셋으로 확장(공격력/체력/방어력). 각 버튼에 현재 능력치, 강화 후 능력치, 필요 골드 표시. 보유 골드는 상단에 있으므로 버튼에서는 생략 가능, 비용은 반드시 표시.

### 9. 모바일 배치

세 버튼을 하단에 세로 배치. 540×960 밖으로 나가지 않고, 겹치거나 글자 잘리지 않으며, 손가락으로 구분해 누를 크기 유지. 전투 영역·경험치 바를 가리지 않음. 새 화면/팝업 없음.

### 10. 버튼 활성화

각 버튼은 자기 비용 기준으로 독립 활성화. 강화 후 골드 감소 시 세 버튼 즉시 재계산. 골드 부족 시 눌러도 변화 없음.

### 11. 상태 표시

상단에 방어력 추가(공격력·방어력 같은 줄 가능). 최대 체력은 기존 HP 표시로 확인.

### 12. 실제 피해량 표시

공격 시 대상 근처에 실제 피해 숫자를 약 0.3~0.6초 표시 후 사라짐(위로 이동/투명). 전투를 막지 않고 같은 공격이 중복 표시되지 않음. 숫자만 사용 가능(예 -10, -4, -3). 치명타·색상 등급·파티클 없음.

### 13. 한글 폰트

새 버튼(체력/방어력)과 방어력 상태 문구에 기존 폰트 처리 적용. 폰트 파일은 커밋하지 않음. 피해 숫자는 한글 폰트 불필요.

---

## 성장 수식

```text
현재 공격력 = 시작 공격력 + 공격력 강화 횟수 × 2 + 레벨업 횟수 × 1
현재 최대 체력 = 시작 최대 체력 + 체력 강화 횟수 × 20 + 레벨업 횟수 × 5
현재 방어력 = 시작 방어력 2 + 방어력 강화 횟수 × 1
```

## 자동 검증

기존 TASK 001·002·003 검증을 삭제하지 않고 계속 통과해야 한다.

추가 검증.

- 피해 계산: 적 공격력 6 - 방어력 2 = 4. 방어력 1회 강화 후 6 - 3 = 3. 최소 피해(공격력 4 - 방어력 10 = 1).
- 체력 강화 1회: 골드 -비용, 최대 체력 +20, 현재 체력 +20(새 최대 초과 금지, 완전 회복 아님), 다음 비용 ×1.4 반올림, 강화 횟수 +1.
- 방어력 강화 1회: 골드 -비용, 방어력 +1, 일반 적 피해 4 → 3, 다음 비용 ×1.4 반올림, 강화 횟수 +1.
- 강화 검증 순서(검증 전용): 체력 1회 → 방어력 1회 → 공격력 2회 이상 → 기존보다 적은 타격으로 처치. 일반 플레이에서는 순서를 강제하지 않는다.
- 불변식: 각 처치 후 공격력·최대 체력·방어력이 위 수식과 일치. 다르면 실제/예상 값을 로그로 출력하고 종료 코드 1.

통과 조건(모두 충족 시 종료 0): TASK 002 공격력, TASK 003 경험치·레벨업, 체력 1회, 방어력 1회, 방어력 전후 피해, 세 능력치 누적 수식. 조기 종료 금지, 무한 실행 방지 제한 유지, 실패 시 어긋난 값 로그.

## 완료 조건

기존 자동전투·골드·경험치·레벨업 정상. 세 강화 버튼 표시·세로 겹침 없음·글자 안 잘림. 골드 부족 버튼 비활성. 세 비용 독립 증가. 체력 강화 시 최대·현재 체력 각각 +20(완전 회복 아님). 방어력 강화 시 +1. 기본 방어력 도입 후 일반 적 실제 피해 4 유지, 1회 강화 후 3. 모든 공격 최소 1 피해. 실제 피해 숫자 표시. 상태에서 공격력·최대 체력·방어력 확인. 레벨 체력/공격력과 강화 체력/공격력 함께 누적. 사망 재시작 루프 유지. 헤드리스 종료 0, 반복 오류·경고 없음. 범위 밖 기능 없음.

## 검증 방법

자동 검증.

```text
"C:\Users\USER\godot-engine\Godot_v4.7-stable_win64_console.exe" --headless --path "C:\Users\USER\unknown" -- --verify
```

PC 화면 검증(가능하면): 세 버튼 겹침/글자 잘림 없음, 피해 숫자 공격마다 1회, 체력 강화 후 최대·현재 체력 즉시 증가, 방어력 강화 후 적 피해 4→3, 기존 전투·레벨업 정상. 못 했으면 미검증으로 기록.

아이폰 사용자 검증: 세 버튼 정확히 누름, 겹침/잘림 없음, 공격력=빨리 잡기·체력=오래 버티기·방어력=피해 감소 체감, 골드 사용처 고민, 버튼 셋이어도 복잡하지 않음. 사용자 확인 전 아이폰 검증 완료로 기록하지 않는다.

## 재미 확인 질문

1. 공격력·체력·방어력의 역할이 다르게 느껴지는가.
2. 골드를 어디에 올릴지 선택하는 재미가 있는가.
3. 체력·방어력 강화가 숫자만 바뀌는 것처럼 느껴지지 않는가.
4. 공격력만 올리는 것이 항상 정답처럼 보이지 않는가.
5. 버튼 셋이어도 조작이 번거롭지 않은가.
