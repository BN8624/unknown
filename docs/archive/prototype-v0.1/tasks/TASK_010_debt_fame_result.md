# TASK 010 — 빚 상환·명성 상승과 보스 결과 화면

## 작업 목적

철퇴의 브루노를 처치했을 때 단순히 골드와 경험치만 얻는 것이 아니라, 게임의 핵심 목표인 빚 상환과 명성 상승이 진행되게 한다.

이번 작업인 보상 획득 → 빚 500골드 자동 상환 → 명성 등급 상승 → 결과 화면에서 성장 의미 확인 → 첫 지역 완료 후 일반 사냥 복귀.

핵심 질문은 다음과 같다.

> 보스 처치가 단순한 전투 승리가 아니라, 빚을 갚고 무명에서 벗어나는 의미 있는 성장으로 느껴지는가.

이번 작업에서는 빚과 명성의 첫 단계만 구현한다.

저장과 불러오기, 다음 지역, 추가 명성 등급은 구현하지 않는다.

## 정본 보강

`UNKNOWN.md`의 빚과 명성 항목에 프로토타입용 시작값을 추가한다.

- 프로토타입 시작 빚: 10,000골드.
- 철퇴의 브루노 처치 시 자동 상환: 500골드.
- 첫 명성 상승: 무명 용병 → 풋내기 용병.

기존 원칙은 유지한다.

- 빚 자동 상환은 플레이어의 보유 골드와 별도다.
- 보스 처치로 받은 개인 골드에서 500골드를 차감하지 않는다.
- 빚 상환은 손실이 아니라 별도의 진행 보상으로 표현한다.

## 빚 상태

- 총 빚: 10,000골드.
- 누적 상환액: 0골드.
- 남은 빚: 10,000골드.

```gdscript
const STARTING_DEBT := 10_000
const BRUNO_DEBT_PAYMENT := 500

var debt_remaining := STARTING_DEBT
var debt_paid_total := 0
```

남은 빚은 0보다 작아지지 않게 한다.

```text
실제 상환액 = min(예정 상환액, 현재 남은 빚)
```

## 명성 상태

시작 등급은 무명 용병, 브루노 첫 처치 시 풋내기 용병.

```gdscript
var fame_rank := "무명 용병"
var fame_advanced := false
```

명성 점수나 여러 등급 자동 계산식은 만들지 않는다. 첫 등급 상승 하나만 구현한다.

## 메인 화면 표시

- 남은 빚을 천 단위 구분 기호로 표시한다(10,000 / 9,500).
- 현재 명성을 표시한다(무명 용병 / 풋내기 용병).
- 기존 레벨·경험치·골드·능력치 표시와 겹치지 않게 한다.

## 브루노 처치 정산

기존 TASK 009 보상에 이어 다음 정산을 한 번만 실행한다.

- 기존 개인 보상 유지: 골드 +200, 경험치 +100. (중복 추가 금지)
- 빚 자동 상환 +500(보유 골드 비차감): 남은 빚 10,000 → 9,500, 누적 상환 0 → 500.
- 명성 상승: 무명 용병 → 풋내기 용병(정확히 한 번).

중복 정산 방지:

```gdscript
var bruno_settlement_done := false
```

정산 완료 후 `bruno_settlement_done = true`. 두 번 호출돼도 골드/경험치/빚/명성/팝업이 중복되지 않는다.

## 보스 결과 화면

개인 보상·빚 자동 상환·명성 상승·첫 지역 클리어를 하나의 결과 패널에 약 3~4초 표시한다. 별도 확인 버튼 없이 자동 복귀한다. 패널이 열린 동안 다음 적을 생성하지 않고 공격도 중단 상태다.

## 패배와의 구분

브루노 패배(생존 부족/화력 부족) 시 빚·명성·정산은 변하지 않고, 보상도 없다. 패배 후 기존 규칙대로 첫 늑대로 복귀·재도전한다.

## 게임 재실행

저장이 없으므로 재실행 시 빚 10,000·누적 0·명성 무명·브루노 미처치로 초기화된다. 저장은 TASK 011에서 구현한다.

## 자동 검증

1. 기존 TASK 001~009 검증.
2. 초기 빚·명성 상태(빚 10,000·누적 0·무명·정산 미완료).
3. 패배 시 정산 없음.
4. 브루노 처치 개인 보상 유지(골드+200·경험치+100).
5. 빚 500 자동 상환(남은 빚 9,500·누적 500).
6. 플레이어 골드 비차감.
7. 명성 무명 → 풋내기 상승.
8. 중복 정산 방지(두 번 호출해도 1회분).
9. 빚 0 하한 처리(빚 300에 500 상환 → 실제 300·남은 0).
10. 메인 UI 텍스트 갱신.

실패 시 예상값과 실제값을 출력하고 종료 코드 1로 끝낸다.
