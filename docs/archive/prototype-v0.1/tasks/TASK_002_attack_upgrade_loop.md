# TASK 002 — 골드 강화 성장 루프 구현

## 작업 목적

단순히 골드 숫자만 추가하지 않는다.

다음 흐름을 한 묶음으로 만들어 첫 번째 성장 재미를 검증한다.

```text
적 처치
→ 골드 획득
→ 공격력 강화
→ 적 처치 속도 단축
```

TASK_001이 "전투를 볼 수 있는가"를 확인했다면, TASK_002는 다음을 확인한다.

> 골드를 모아 공격력을 올리고, 적을 더 빨리 잡는 과정이 작은 만족감을 주는가.

---

## 먼저 읽을 문서

1. `CLAUDE.md`
2. `UNKNOWN.md`
3. `HANDOFF.md`
4. `checklist.md`
5. `context-notes.md`

`UNKNOWN.md`와 `CLAUDE.md`는 이번 작업에서 수정하지 않는다.

기존 `Battle.gd`의 전투 루프 위에 최소 변경으로 얹는다.

---

## 구현 범위

작게 제한한다.

* 적 처치 시 골드 5 획득.
* 현재 골드 표시.
* 공격력 강화 버튼 하나.
* 첫 강화 비용 20골드.
* 강화 시 공격력 +2.
* 다음 강화 비용은 직전 비용 × 1.4.
* 현재 공격력, 강화 후 공격력, 강화 비용 표시.
* 강화 전후로 적 처치 속도(처치당 타격 수) 차이를 눈으로 확인할 수 있게 한다.

밸런스 수치는 `Battle.gd` 상단 상수 한 곳에서 조정 가능하게 둔다.

---

## 구현하지 말아야 할 것

* 체력 강화.
* 방어력 강화.
* 경험치.
* 레벨업.
* 특성.
* 엘리트와 보스.
* 저장.
* 상점, 여러 강화 항목.

좋은 아이디어가 떠올라도 이번 구현에는 넣지 않는다.

필요하면 `context-notes.md`의 보류 항목에 한 줄로만 남긴다.

---

## 완료 조건

* 프로젝트가 오류 없이 실행된다.
* 적을 처치하면 골드가 5씩 늘어난다.
* 현재 골드가 화면에 표시된다.
* 공격력 강화 버튼이 보이고 누를 수 있다.
* 골드가 충분할 때만 강화된다.
* 강화 시 골드가 비용만큼 줄고 공격력이 +2 된다.
* 다음 비용이 직전 비용의 약 1.4배가 된다.
* 현재 공격력, 강화 후 공격력, 비용이 표시된다.
* 공격력을 올리면 같은 적을 더 적은 타격으로 잡는다.
* TASK_001의 전투 루프가 그대로 유지된다.
* 콘솔에 반복 오류가 없다.

---

## 검증 방법

1. 프로젝트를 실행한다.
2. 적을 여러 마리 잡아 골드를 모은다.
3. 공격력 강화 버튼을 눌러 공격력을 올린다.
4. 강화 전후로 적을 잡는 데 걸리는 타격 수가 줄어드는지 확인한다.
5. 콘솔에 반복 오류가 없는지 확인한다.

실제로 실행하지 못한 항목은 검증 완료로 표시하지 않는다.

타격감과 만족감은 사용자 플레이로 판단한다.

---

## 문서 갱신

* `checklist.md`: 이번 완료 항목 체크.
* `HANDOFF.md`: 현재 목표, 완료 내용, 실행 방법, 검증 결과, 남은 위험, 다음 액션 갱신.
* `context-notes.md`: 새 결정이나 보류 아이디어가 있을 때만 기록.

---

## 작업 종료 지점

골드 강화 성장 루프를 구현하고 검증한 뒤 종료한다.

사용자가 직접 "강해지는 맛"을 확인하기 전까지 체력·방어력·경험치·레벨·특성으로 넘어가지 않는다.
