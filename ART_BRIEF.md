# 균열기사 — 아트 외주 발주서 (Art Brief)

세로형 모바일 방치형 RPG. 이 문서를 아티스트에게 그대로 전달하면 된다.
**스타일 레퍼런스(필수)**: 이미 보유한 AI 생성 3종(타이틀 씬·보스 초상화·아이콘 8종) = `assets/ui/`. "이 톤·디테일을 유지/상향"이라고 명시할 것.

---

## 1. 게임 개요 (아티스트용 한 장 요약)

- 장르: 세로 화면(540×960) 자동전투 방치형 RPG. 던전을 끝없이 오르며 성장.
- 분위기: **다크 판타지 픽셀아트**. 심해 인디고 + 보라 균열 에너지 + 금색 강조. 횃불 켜진 돌 던전.
- 화면 구성: 상단 HUD / 중앙 횡전투(좌=기사, 우=적) / 하단 성장 패널.
- 핵심 정서: "균열에서 쏟아지는 마물을 홀로 베어내는 기사."

## 2. 아트 스타일 규격 (전 자산 공통)

- **스타일**: 고해상도 픽셀아트(보유 AI 아트 수준의 디테일). 또는 합의 시 일러스트형 가능.
- **포맷**: PNG, **RGBA 투명배경**(체커·흰배경 금지), 8bit/채널.
- **팔레트**: 어두운 보라/인디고 베이스, 보라 균열 발광, 금색 포인트, 지역별 변주(1지역 회청 벽돌 / 2지역 화염 적·주황 / 3지역 얼음 청록).
- **해상도(중요)**: 픽셀아트면 **네이티브 작게 + 정수배 스케일** 권장. 캐릭터 네이티브 ~**64×64**(보스 ~96×96), 배경 **540×960(@1x) + 1080×1920(@2x)** 동시 납품. 일러스트형이면 캐릭터 512px+.
- **방향**: 횡전투 기준 — **주인공은 오른쪽을 바라봄, 적은 왼쪽을 바라봄**(또는 정면 3/4 뷰 통일). 피벗/기준점은 **발 바닥 중앙**(접지용).
- **여백**: 캐릭터는 캔버스 가장자리에서 1~2px 여백(외곽선 처리 여유).
- **납품 단위**: 자산별 **개별 PNG** + (애니는) 가로 스프라이트 시트 + 프레임 수·FPS 명기. PSD/레이어 원본 동봉 권장(수정·패럴랙스용).

## 4. 캐릭터 세부 (게임 내 표시 크기·이름)

표시 크기(코드 기준, 참고용): 주인공 ~104px, 일반 적 ~70~86px, 보스 ~140~184px. 보스 초상화 ~340px.

**일반 적 (지역별, 실루엣 공유 가능)**
- 1지역 갈라진 변경: 들개, 부패 정령, 균열 고블린, 가시벌레(거미), 그림자 망령
- 2지역 잿빛 협곡(화염): 잿불 박쥐, 용암 슬라임, 그을린 늑대, 재 정령, 불씨 골렘
- 3지역 얼어붙은 심연(얼음): 서리 망령, 얼음 슬라임, 동토 늑대, 심연 술사, 한기 거미

**보스 (지역당 4, 초상화+전투 스프라이트)**
- 1지역: 거대 거미 아라크, 이끼 골렘 모스, 망령군주 베일, 균열의 수호자
- 2지역: 화염군주 이그, 용암 거인 칼, 잿빛 드레이크, 협곡의 지배자
- 3지역: 빙결군주 헬름, 심연 거미 여왕, 얼어붙은 망령왕, 심연의 지배자
- (예산상 축소 시: 보스 실루엣 6종 + 지역 팔레트 변주로 12종 커버)

## 5. UI 키트 세부

- **아이콘**(개별 PNG, 128~256px, 금테 프레임 통일): 공격력(검)·체력(하트)·방어력(방패)·치명타(단검)·골드(금화)·균열석(보라 크리스탈)·스킬(해골불꽃)·시간(모래시계)·경험치·레벨·환생·임무·업적·설정·잠금. (8종은 보유, 7종 추가)
- **패널/창 프레임**: 9슬라이스용(모서리 안전영역 표시). 일반/강조 2종.
- **버튼**: 9슬라이스 + 상태 3종(기본/눌림/비활성). 일반·강조(금테)·위험(적색) 변형.
- **바**: 체력바·경험치바·진행바(테두리+채움 분리).
- **스킬 버튼 원형 프레임** 2종(강타·가속).

## 6. 발주·납품 체크리스트 (이대로 요청)

1. 위 **스타일 규격(2장)**과 **레퍼런스(보유 AI 아트)** 먼저 합의 → 테스트로 **주인공 1티어 + 적 1종 + 배경 1장** 선납품받아 톤 확인 후 본작업.
2. 모든 캐릭터: **투명 RGBA, 발 바닥 중앙 피벗, 일관된 캔버스 크기**(애니 프레임 동일 크기).
3. 애니: **가로 스프라이트 시트 + 프레임수/FPS 텍스트**, 또는 개별 프레임 PNG(0001…).
4. 배경: **레이어 분리 PNG(또는 PSD)** + 합본 미리보기 1장.
5. 파일명: 아래 **네이밍 규칙**대로(엔진 드롭인용).
6. 라이선스: **상업 사용·수정·재배포 가능(전권 양도 권장)**, 출처/저작권 서면 명시.
7. 납품: 원본(PSD/Aseprite) + 내보낸 PNG + (필요시) 스프라이트 시트.

## 7. 엔진 드롭인 네이밍 규칙 (이대로 받으면 바로 적용)

- 주인공: `hero_t1.png`(+`_t2`,`_t3`), 애니 시트 `hero_t1_attack.png` 등.
- 일반 적: `enemy_<영문키>.png` (예: `enemy_wolf`, `enemy_slime`, `enemy_spider`…).
- 보스 스프라이트: `boss_<영문키>.png` / 초상화: `portrait_<영문키>.png`.
- 배경: `bg_region1.png`(원경/벽/바닥 레이어는 `_far/_wall/_floor` 접미사) … `bg_region3`.
- 아이콘: `icon_<키>.png`(atk/hp/def/crit/gold/crystal/skull/time/exp/level/prestige/mission/ach/settings…).
- UI 프레임/버튼/바: `ui_panel.png`, `ui_btn_normal/pressed/disabled.png`, `ui_bar_*.png`.

> 영문 키 매핑이 필요하면 개발자(나)에게 요청 — 코드 기준으로 정확히 발급해 줌.

- 방식 B(권장): 12 보스 각각 고유 초상화(`portrait_<보스영문>`), 임팩트 최대. 영문 키는 요청 시 발급.

- 일반 적 15 고유(`enemy_<지역><번호>` 또는 의미 영문) + 보스 12 고유 + 초상화 12.

## 부록 B. 예산 규모별 발주안 (3단계)

### 💧 최소(Lite) — "지금 게임을 즉시 업그레이드"
- 주인공 3티어(정지 only, 애니 X)
- 적 공유 베이스 **7종** + 보스 베이스 **6종**(정지) + 보스 초상화 **6**
- 지역 배경 **3**(레이어 분리)
- UI: 패널/버튼 9슬라이스 1세트 + 부족 아이콘 7종 + 로고 1
- → **드롭인 즉시 반영. 가장 가성비.** 현 코드 구조 그대로.

### 🔷 권장(Standard) — "출시 퀄리티"
- 주인공 3티어 × **애니 4세트**(idle/attack/hit/death)
- 적 7~10종 + **가벼운 애니(idle/hit)**
- 보스 6~8종(애니 idle/attack/hit/death) + **초상화 8~12**
- 지역 배경 3(레이어+패럴랙스) + 이펙트 시트 일부(베기·타격·보스등장)
- UI 풀 키트(상태별 버튼·아이콘 15·진행바) + 로고/스플래시
- → **체감 품질 최상. 전투가 살아남.**

### 💠 풀(Premium) — "확장까지"
- Standard + **네임드별 고유 적 15·보스 12·초상화 12**
- 4지역 이상 배경/적/보스 세트, 펫/탈것, 추가 외형 티어
- 전체 이펙트 시트 6종, 정식 UI 모션용 에셋
- → 장기 라이브 대비.

> 견적 받을 때: 위 **수량 + 부록 A 표 + 2장 스타일 규격**을 통째로 전달하면 정확한 견적이 나온다. 최소안부터 시작해 반응 보고 확장 권장.

## 8. 예산 가늠용 총량

- P0: 캐릭터 3티어×4애니 + 보스 초상화 6~12 + 배경 3(레이어).
- P1: 일반 적 10~15(+간단 애니) + 보스 스프라이트 6~12 + UI 키트 1 + 로고.
- 마켓 견적 시 위 **수량·해상도·애니 프레임 수**를 그대로 전달하면 정확한 견적이 나온다.
