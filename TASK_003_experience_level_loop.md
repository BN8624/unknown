# TASK 003 — 경험치·레벨업 성장 루프 구현

## 작업 목적

기존 자동전투와 골드 강화 루프 위에 경험치와 레벨업을 추가한다.

이번 작업에서 검증할 흐름은 다음과 같다.

```text
적 처치
→ 골드와 경험치 획득
→ 경험치 누적
→ 레벨업
→ 공격력과 최대 체력 증가
→ 체력 완전 회복
→ 이전보다 조금 강해진 상태로 전투 지속
```

TASK 001은 자동전투를 지켜볼 수 있는지 확인했다.

TASK 002는 골드를 모아 공격력을 강화하는 과정에서 작은 성장 만족감이 생기는지 확인했다.

TASK 003은 다음을 확인한다.

> 적을 계속 처치해 레벨이 오르고, 캐릭터가 자동으로 조금씩 강해지는 과정이 성장 동기를 만드는가.

기존 `Battle.gd`의 자동전투와 골드 강화 구조를 전면 재작성하지 않는다. 최소 변경으로 경험치와 레벨업을 추가한다.

---

## 현재 기준 (그대로 유지)

용병 자동 전진, 적 자동 등장, 자동 공격, 처치 후 반복, 처치 시 골드 5, 공격력 강화 버튼(+2, 비용 증가), 사망 후 자동 회복·재시작, 헤드리스 `--verify`, Web 내보내기, 한글 폰트 적용.

기존 기능을 삭제하거나 동작을 바꾸지 않는다.

---

## 구현 범위

### 1. 경험치 상태

시작 레벨 1, 시작 경험치 0, 적 처치당 경험치 5.

밸런스 수치는 `Battle.gd` 상단 상수 영역에 모은다.

```text
const EXP_PER_KILL := 5
const START_LEVEL := 1
const LEVEL_ATK_GAIN := 1
const LEVEL_HP_GAIN := 5
```

### 2. 다음 레벨 필요 경험치

`UNKNOWN.md` 규칙을 따른다.

```text
필요 경험치 = 20 + 현재 레벨 × 5
```

레벨 1→2: 25, 2→3: 30, 3→4: 35. 현재 레벨은 레벨업하기 전의 레벨이다. 작은 함수 하나로 계산한다.

### 3. 적 처치 시 경험치 획득

```text
적 처치 확정 → 골드 +5 → 경험치 +5 → 레벨업 조건 확인 → UI 갱신 → 적 사망 및 다음 전투
```

골드와 경험치 중 하나도 누락되지 않고, 한 마리가 중복 지급하지 않는다.

### 4. 레벨업 처리

현재 경험치가 필요 경험치 이상이면 레벨업한다. 레벨 +1, 공격력 +1, 최대 체력 +5, 현재 체력을 새 최대 체력까지 완전 회복. 핵심은 레벨업 직전 공격력에서 정확히 +1.

### 5. 골드 강화와 레벨 성장의 합산

```text
현재 공격력 = 시작 공격력 + 골드 강화 횟수 × 2 + 레벨업 횟수 × 1
```

레벨업이 골드 강화 공격력을 초기화하지 않는다. 강화 버튼에는 레벨업으로 바뀐 현재 공격력이 즉시 반영된다.

### 6. 초과 경험치 처리

경험치가 필요량을 넘으면 버리지 않는다. 필요 경험치를 차감하고 레벨을 올리며 남은 경험치를 유지한다. 필요하면 "현재 경험치 ≥ 필요 경험치인 동안 반복". 레벨 제한과 최고 레벨은 만들지 않는다.

### 7. 레벨과 경험치 UI

상단에 현재 레벨, 현재 경험치, 다음 필요 경험치를 표시한다. 기존 골드·처치·용병 체력·적 체력 표시는 유지한다. 한 줄이 복잡하면 두 줄로 나눈다.

```text
레벨 1  EXP 15/25  골드 20  처치 3
용병 HP 72/100  적 HP 20
```

### 8. 경험치 진행 바

현재 경험치 비율을 보여주는 간단한 바. 레벨업하면 새 기준으로 초기화. 체력 바와 혼동되지 않는 위치, 다른 UI와 겹치지 않게. 장식·애니메이션 없음.

### 9. 레벨업 표현

- 레벨업 문구: "레벨 업! N"을 약 0.8~1초 표시. 숫자는 현재 레벨에 맞춤. 전투·버튼 입력을 막지 않는다.
- 용병 강조: 짧게 커졌다 복귀, 또는 짧게 밝아짐. 복잡한 파티클은 없음.

### 10. 레벨업과 체력 회복

레벨업 시 현재 체력을 새 최대 체력까지 완전 회복하고 체력 바도 즉시 가득 찬다. 처치/전투/시간/흡수 회복은 추가하지 않는다. 레벨업 완전 회복만 추가하고, 기존 사망 후 전체 회복 안전장치는 유지한다.

### 11. 한글 폰트 적용

레벨 표시, 경험치 표시, 레벨업 문구에도 기존 한글 폰트 처리를 적용한다. Web 빌드에서 한글이 깨지지 않아야 한다. 폰트 파일은 저장소에 새로 커밋하지 않는다.

## 자동 검증

기존 `--verify`를 삭제하지 않는다. TASK 002 항목(골드 +5, 강화, 비용 증가, 공격력 증가, 루프 반복, 처치 타격 수 변화, 종료 코드 0, 오류·경고 없음)이 계속 통과해야 한다.

추가 검증.

- 적 처치당 경험치 정확히 +5. 로그 예: `[VERIFY] xp kill=1 level=1 exp=5/25`.
- 첫 레벨업: 레벨 1에서 5마리 처치(경험치 25) → 레벨 2. 레벨업 후 레벨 2, 남은 경험치 0, 다음 필요 30, 최대 체력 +5, 현재 체력 = 새 최대 체력, 공격력 = 직전 +1. 골드 강화 공격력은 유지.
- 공격력 계산: `merc.atk > 시작값`만으로 골드 강화 성공을 판단하지 않는다. 강화 횟수(`attack_upgrade_count`)를 명시적으로 추적하고, `현재 공격력 == 시작 + 강화×2 + 레벨업×1`과 일치해야 한다.

종료 조건: TASK 002 검증과 TASK 003 검증이 모두 통과한 뒤 종료 코드 0. 하나만 통과한 상태로 조기 종료하지 않는다. 실패 시 어떤 값이 달랐는지 로그로 남기고 종료 코드 1. 무한 실행 방지용 최대 처치 수/제한은 유지한다.

## 구현하지 말아야 할 것

체력·방어력 골드 강화, 방어력 능력치, 특성 포인트, 레벨 10 보상, 강타·연격·반격, 프리셋, 적 추가, 엘리트, 보스, 장비, 빚, 명성, 저장, 오프라인 보상, 사운드, 사망 연출 개선, 상점, 여러 화면, 레벨 제한, 경험치 배율, 복잡한 성장 데이터, 범용 성장 프레임워크.

레벨 10마다 특성 포인트 규칙은 `UNKNOWN.md`에 있으나 이번에는 구현하지 않는다. 지금은 레벨·경험치 기본 성장 루프만 확인한다.

## 검증 방법

자동 검증.

```text
"C:\Users\USER\godot-engine\Godot_v4.7-stable_win64_console.exe" --headless --path "C:\Users\USER\unknown" -- --verify
```

로그에서 경험치 +5, 레벨 1 필요 25, 5마리 후 레벨 2, 레벨업 후 경험치 0, 다음 필요 30, 공격력 +1, 최대 체력 +5, 현재 체력 완전 회복, 합산 정확성, 기존 TASK 002 통과, 종료 코드 0, 오류·경고 없음을 확인한다.

화면 검증(PC)으로 경험치 바 증가, 레벨업까지 거리, 레벨업 문구·효과, 회복, 강화 버튼 갱신, 기존 강화 버튼 정상, 정보 겹침 없음을 확인한다.

아이폰 Web 화면의 최종 가독성과 성장 체감은 사용자가 직접 판단한다. 사용자 확인 전에는 아이폰 검증 완료로 기록하지 않는다.

## 재미 확인 질문 (사용자)

1. 경험치가 차오르는 모습을 계속 확인하고 싶은가.
2. 레벨업까지 남은 거리가 명확하게 보이는가.
3. 레벨업 순간이 충분히 눈에 띄는가.
4. 공격력과 최대 체력이 오른 것이 전투에서 느껴지는가.
5. 레벨업 완전 회복이 반갑게 느껴지는가.
6. 골드 강화와 자동 레벨 성장이 서로 다른 성장 방식으로 느껴지는가.
