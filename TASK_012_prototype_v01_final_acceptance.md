# TASK 012 — 프로토타입 v0.1 전체 흐름 최종 검증

## 작업 목적

지금까지 구현한 첫 프로토타입을 새 게임 상태부터 처음부터 끝까지 점검한다. 새 기능 추가가 아니라, 다음 전체 흐름이 하나의 게임으로 자연스럽게 이어지는지 확인하는 최종 통합 검증이다.

새 게임 → 자동 전투 → 골드 획득 → 기본 능력치 강화 → 경험치·레벨업 → 레벨 10 특성 선택 → 일반 적 3종 → 오우거 엘리트 → 브루노 보스 → 패배 시 성장 후 재도전 → 보스 처치 → 빚 상환·명성 상승 → 자동 저장 → 새로고침 후 진행 복원.

핵심 질문: 이 프로토타입이 기능들의 집합이 아니라, 처음부터 끝까지 계속 플레이할 수 있는 하나의 방치형 게임으로 느껴지는가.

## 먼저 읽을 문서

CLAUDE.md → UNKNOWN.md → HANDOFF.md → TASK_009 → TASK_010 → TASK_011 → checklist.md → context-notes.md → 현재 Battle.gd. 기존 TASK 001~011 기능·자동 검증을 유지한다.

## 작업 원칙 — 신규 기능 금지

오프라인 보상·다음 지역·새 적/보스·특성 2·3단계·프리셋·장비·상점·퀘스트·환생·추가 명성 등급·추가 빚 상환·사운드·완성형 아트·새 저장 항목·범용 프레임워크·대규모 리팩터링을 추가하지 않는다.

허용되는 변경은 다음뿐이다.
- 전체 흐름을 막는 버그 수정.
- 저장·전투·UI 충돌 수정.
- 아이폰에서 버튼·문구를 못 쓰게 하는 문제 수정.
- 문서와 실제 구현 불일치 수정.
- 검증 중 발견된 명백한 잘못된 수치·상태 전이 수정.

## 문서 상태 동기화

TASK 011 아이폰 확인 완료(저장 지속성·입력 수정)를 HANDOFF.md·checklist.md에 완료로 유지한다.

HANDOFF.md 실행 방법에 남은 잘못된 설명("시작 특성 포인트 임시 3개로 바로 시험") 제거. 실제 규칙: 신규 게임은 특성 포인트 0, 레벨 10에 첫 1포인트. DEBUG_START_TRAIT_POINTS가 문서 어디에도 현재 기능처럼 남지 않게 한다.

## 1. 자동 회귀 검증

`-- --verify` 실행. TASK 001~011 모두 통과, 종료 코드 0, 반복 오류·경고 없음, 검증 저장이 실제 사용자 저장을 안 건드림, 종료 후 테스트 저장 파일 미잔존. 기존 검증을 삭제·완화하지 않고, 새 대규모 하네스를 만들지 않는다. 통합에서 자동 검증이 놓친 명백한 상태 문제가 나오면 그때만 최소 검증 추가.

## 2. 라이브 스모크(헤드리스/PC)

첫 늑대 등장·적 순환·골드/경험치 증가·강화·레벨업·특성 중복 미지급·오우거/브루노 등장·패배 후 첫 늑대 복귀·처치 후 미재등장·결과 패널 후 사냥 재개·저장 과정 런타임 오류 0. 아이폰 전체 검증을 PC로 대신하지 않는다.

## 3. Web 빌드 갱신

최신 코드를 Web 내보내기. 오류 없이 완료·한글 폰트 포함·저장/자격증명 미포함·정적 서버 최신 제공·접속 주소에 버전 값(`?v=12`)으로 캐시 회피 안내.

## 4. 아이폰 최종 검증 준비

저장 초기화(2회 확인)로 신규 상태 확인: 레벨1·경험치0·골드0·강화 0/0/0·포인트0·투자없음·브루노 미처치·빚10,000·누적0·무명. 임시 포인트 3개 지급되면 실패.

## 5. 아이폰 전체 흐름 검증 (사용자)

일반 Safari 탭, 비공개 모드 금지. 구간 A 시작·기본 전투 / B 골드 성장 / C 레벨업 / D 첫 특성(패널·닫기·무료 초기화·저장 초기화 구분, 한 번 터치) / E 일반 적·엘리트 / F 브루노 보스(보스전 중 버튼 작동, 패배 원인, 재도전 체력 초기화) / G 보스 승리 결과(골드/경험치 1회, 골드 비차감, 빚 9,500, 무명→풋내기, 결과 패널, 첫 지역 완료, 미재등장) / H 저장 복원(새로고침 후 전 상태 유지, 보스·결과 패널 미재등장, 완전 회복, 첫 늑대 시작, 입력 정상).

## 6. 플레이 시간·흐름 기록

초 단위 계측 추가 금지. 대략: 첫 강화·레벨 10·오우거 첫 도전·브루노 첫 도전·브루노 처치·전체 완료 시점. 목표 약 15분이나 숫자 맞추려 자동 밸런스 변경 금지. 레벨 10까지 과도/특성 전 반복감/보스 전 흥미 소실/재도전 지연/강화 너무 빠름·느림은 문제로 기록. 사용자 의견 없이 임의 밸런스 조정 금지.

## 7. 최종 재미 판단 (사용자)

10개 질문(자동 전투 지속성·강화 체감·특성 대기 동기·특성 플레이 변화·단계 상승감·재도전 의욕·빚·명성 보상감·저장 후 지속 의향·다음 지역 기대·재플레이 의향). 8~10 긍정=v0.1 승인 / 5~7=핵심 유지·문제만 수정 후 재검증 / 0~4=콘텐츠 추가 말고 핵심 루프 재검토. 숫자 강제 판정 금지, 전체 인상·실제 중단 지점 우선.
